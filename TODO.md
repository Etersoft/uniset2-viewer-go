## Фильтрация датчиков

### Требования к серверной стороне
Для корректной работы фильтрации датчиков UniSet2 API должен поддерживать:

1. **IONotifyController** (`/api/v2/{object}/sensors`)
   - [ ] Параметр `filter` для фильтрации по имени датчика (подстрока)
   - [ ] Параметр `type` для фильтрации по типу (AI, DI, AO, DO)
   - Пример: `GET /api/v2/SharedMemory/sensors?filter=Temp&type=AI&offset=0&limit=100`

2. **OPCUAExchange** (`/api/v2/{object}/sensors`)
   - [x] Параметр `filter` уже поддерживается
   - [ ] Параметр `type` для фильтрации по типу (AI, DI, AO, DO)

### Текущие ограничения
- IONC: фильтрация выполняется локально на загруженной странице, что неэффективно для больших списков
- Для полноценной работы требуется серверная поддержка фильтрации
